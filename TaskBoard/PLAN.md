# 無料前提で作るステップ別計画

## Summary
- まず **Cloudflare Pages + Firebase無料枠** で、Geminiなしで動くWebアプリにする。
- 認証とDBは Firebase、配信は Cloudflare Pages。スマホではPWAとしてホーム画面追加できる形にする。
- ネイティブアプリ化は後段で Capacitor を追加する。ただし App Store公開はApple Developer Program費用が必要なので、無料運用中はPWAまでを本線にする。

## Step 1: Webアプリ化の土台
- Vite + React + TypeScript + Tailwind の新規プロジェクトを作る。
- 既存TSXを `App.tsx` に移植し、Gemini専用変数を削除する。
- Firebase設定は `.env.local` から読む。
- Cloudflare Pagesへデプロイできる構成にする。
- ここでの完成条件:
  - ローカルで起動できる。
  - Cloudflare Pages上で画面が表示される。
  - Geminiを開かなくても使える。

## Step 2: Firebase認証と安全な保存
- Firebase AuthenticationでGoogleログインを有効化する。
- Firestoreにタスク、ボード、設定を保存する。
- Firestore Rulesで許可ユーザーだけ読み書き可能にする。
- 初期は「あなた」と「奥さん」のGoogleメールアドレスだけ許可する。
- 未ログイン時はログイン画面を表示する。
- ここでの完成条件:
  - 許可アカウントだけログイン後に使える。
  - 未許可アカウントではデータを読めない・書けない。
  - タスク追加、編集、削除がCloudflare上で同期される。

## Step 3: ボード共有・権限管理
- ボードに `ownerId`, `memberIds`, `visibility` を持たせる。
- ボードは初期状態ではプライベート。
- オーナーだけがメンバー追加・削除・ボード削除できる。
- 参加中ボードと自分のボードをUI上で区別する。
- 招待は最初は「メールアドレスで追加」方式にする。
- ここでの完成条件:
  - ボードごとに共有相手を変えられる。
  - 共有されていないボードは他人から見えない。
  - オーナー以外はボード削除できない。

## Step 4: スマホ向けUI改善
- 下部タブを追加する。
  - ボード
  - カレンダー
  - 検索
  - 設定
- OS依存のプルダウンを、独自の選択UIへ置き換える。
- Android Chromeのドラッグ問題対策として、ドラッグ以外の移動手段を追加する。
  - タスクカード内にステータス変更ボタン/メニューを用意。
  - ステータス・担当者の並び替えは長押し対応を検討。
- 縦スクロール対策として、ステータスごとの折りたたみを標準化する。
- ここでの完成条件:
  - スマホでドラッグなしでもタスク移動できる。
  - 長いボードでもステータス単位で見やすい。
  - 主要操作が親指で届く。

## Step 5: カレンダー機能
- カレンダータブを作る。
- 期日ありタスクを日付ごとに表示する。
- 期日なしタスクは専用アイコンから一覧表示する。
- カレンダーに表示する対象ボードを選べるようにする。
- 期日が近いタスクを上部に目立たせる。
- ここでの完成条件:
  - 今日・今週・月表示で期日タスクを確認できる。
  - 期日なしタスクを見落とさない。
  - 複数ボードから表示対象を選べる。

## Step 6: タスク機能強化
- 担当者を複数選択できるようにする。
- サブタスク編集をポップアップ化する。
- サブタスク進捗を「完了数/総数」とプログレスバーで表示する。
- サブタスク削除確認を追加する。
- タスク追加日時、完了日時を保存・表示する。
- タスク検索フィルタを追加する。
- メモ欄は簡易Markdown表示に対応する。
- ここでの完成条件:
  - 複数担当者で絞り込みできる。
  - サブタスク操作の誤操作が減る。
  - 文字検索で目的のタスクにすぐ辿り着ける。

## Step 7: データ管理・アーカイブ
- 右上歯車メニューに以下を集約する。
  - 担当者管理
  - ステータス管理
  - データ管理
  - ご意見箱リンク
- 完了タスクを一括でゴミ箱へ移動できるようにする。
- ゴミ箱を一括で空にできるようにする。
- 完了タスクのアーカイブ機能を追加する。
- バックアップ復元は「上書き」ではなく、確認付きで「追加/上書き」を選べるようにする。
- ここでの完成条件:
  - 完了済みタスクを安全に整理できる。
  - 復元時に意図せず既存データを壊さない。
  - 設定系の導線が歯車メニューにまとまる。

## Step 8: PWA対応
- Web App Manifestを追加する。
- スマホのホーム画面に追加した時、アプリ風に起動するようにする。
- アイコン、テーマカラー、スプラッシュ表示を設定する。
- オフライン時は「接続できません」画面を表示する。
- ここでの完成条件:
  - iPhone/Androidのホーム画面から起動できる。
  - ブラウザ感を減らしてアプリっぽく使える。
  - 無料のまま運用できる。

## Step 9: 通知・繰り返し
- 期日が近いタスクの通知を追加する。
- まずはアプリ内通知から始める。
- その後、必要ならWeb Push通知を検討する。
- 繰り返しタスクは、通常タスクとは別に `recurrence` 設定を持たせる。
- 習慣・日課用ボードは、通常ボードの派生として実装する。
- ここでの完成条件:
  - 今日/明日/期限切れのタスクに気づける。
  - 日課や定期タスクを手動で毎回作らなくてよい。

## Step 10: Capacitorでスマホアプリ化
- 既存ReactアプリにCapacitorを追加する。
- iOS/Androidプロジェクトを生成する。
- Firebase AuthがモバイルWebView内で動くか確認する。
- アプリ内通知など、PWAでは不足する機能があればCapacitor Pluginで補う。
- 無料運用の間はストア公開せず、開発端末で検証する。
- ここでの完成条件:
  - 同じReactコードをスマホアプリの箱で動かせる。
  - 将来ストア公開できる土台がある。
  - Web版/PWA版を壊さずに維持できる。

## Test Plan
- 各ステップごとに `npm run build` が通ることを確認する。
- Cloudflare Pagesの本番URLでスマホ表示を確認する。
- Firebase Rulesで許可ユーザー/未許可ユーザーの読み書きを検証する。
- Android Chrome、iPhone Safari、PC Chromeで主要操作を確認する。
- タスク追加、編集、削除、復元、共有、カレンダー表示、検索を手動テストする。

## Assumptions
- 無料運用を最優先する。
- 初期公開先は Cloudflare Pages。
- 認証とDBは Firebase無料枠。
- 最初のログインはGoogleのみ。LINEログインは後回し。
- 既存Gemini上のデータは移行せず、手入力で作り直す。
- ネイティブアプリ化は将来対応。まずはPWAまでを無料で完成させる。
